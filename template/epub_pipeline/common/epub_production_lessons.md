# EPUB 制作质量经验 / EPUB Production Quality Lessons

## 来源 / Source

本文件沉淀自《黑人北极探险家》制作过程中的问题、返工和用户反馈，作为后续所有 EPUB 项目的通用质量规则。

## 字体 / Fonts

- 默认不要在 EPUB CSS 中写死 `font-family`。
- 如果阅读器尊重出版商字体，写死字体会导致用户无法在 App 中切换字体。
- 不能因为某个字体好看就直接嵌入完整中文字体；完整 CJK 字体通常几十 MB，会显著膨胀 EPUB。
- 若必须指定字体，使用字体子集化，并记录授权、体积和 fallback。
- `霞鹜文楷/LXGW WenKai` 可作为审美方向，但不应完整塞入普通 EPUB。

## 封面 / Cover

- EPUB 必须有封面页和 OPF `cover-image`。
- 封面图片应压缩到合理体积。一本 280KB 正文的书，不应使用 3MB 未压缩封面。
- 推荐 EPUB 内封面使用 JPG，保留高清 PNG/PSD 等作为工程素材，不直接打包。
- 封面文字需要人工或确定性排版核验，图像模型不可靠地处理精确居中。

## Metadata 与版本说明 / Metadata and Book Info

- 读者看到的书籍详情首先应是本项目版本信息，不只是原书信息。
- 必须包含：`LifeBook 书坊`、译制时间、公版来源 URL、公版说明、作者信息、原书信息、本书简介。
- 品牌名必须全书一致，不得出现旧名称残留，例如 `LifeBook 翻译组`。

## 标题与排版 / Headings and Layout

- 手机窄屏下章节标题不得过大。
- `第X章` 和章节说明的字号要一致或视觉协调，不能一个小一个大。
- 正文行距、缩进、段落间距要适合中文长文阅读。
- 样章阶段必须检查标题、正文、封面、版本说明在阅读器中的实际效果。

## 评审 / Review

- 主执行 AI 容易忽略自己刚做过的错误，必须有独立评审。
- 双 Agent 评审必须分工：一个看翻译内容，一个看 EPUB 工程和排版。
- 评审发现 P0/P1/P2 问题时，必须回退到对应阶段，不得只解释。
