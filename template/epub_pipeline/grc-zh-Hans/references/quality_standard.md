# 古希腊文到简体中文译文质量标准 / Ancient Greek To Simplified Chinese Quality Standard

本文件是 `template/epub_pipeline/targets/zh-Hans/quality_framework/` 的简体中文目标语言规则在古希腊文源文本场景下的应用摘要。更完整的中文目标语言质量规则见目标语言质量框架；古希腊文源语言问题见 `references/ancient_greek_source_notes.md`。

## 总目标

译文要像严谨中文译者从古希腊文底本翻出的书，而不是现代第二语言译本的中文转述。

## 五维质量

1. 准确：事实、论证、语法关系、专名、数字和文本不确定性不误。
2. 通达：中文自然，不保留希腊文句法外壳。
3. 可追溯：重要疑难处能追回希腊文底本、版本和参考证据。
4. 术语稳定：专名、术语、公式化表达和技术概念全书一致。
5. 可出版：注释、目录、EPUB 格式、图表和校验记录稳定。

## 古典文本翻译边界

允许：

- 为中文理解补出必要主语、指代或逻辑关系。
- 把希腊文长句拆成自然中文句群。
- 用译注说明异文、历史背景、术语传统或现代读者可能误解的概念。

不允许：

- 直接从现代英译、法译、德译等第二语言译本转译。
- 为了顺畅而隐藏异文、残损、拟补或语法不确定。
- 把古典概念强行现代化，导致读者误以为原文已有现代科学或现代制度含义。
- 复制仍受版权保护参考译本的表达、注释或结构。

## 一票否决

以下任一出现，章节不得进入 `chapters/final/`：

- 漏译整段或重要论证步骤。
- 专名、地点、年代、数字、术语或指代关系错误。
- 参考译本转译痕迹明显。
- 未记录关键异文或 OCR/转写不确定处。
- 中文明显不通顺，或保留希腊文句法外壳。
- 技术类作品启用 profile 后，未通过技术审计或图表/表格审计。
