# 古希腊文到简体中文翻译补充说明 / Ancient Greek to Chinese Translation Notes

## 基本判断 / Basic Judgment

古希腊文到简体中文的 EPUB 不能只满足“意思大概对”和“文件能打开”。中文读者最终读到的应是一本来源清楚、版本可追溯、中文自然、译注适度、排版完成的书。

An Ancient-Greek-to-Chinese EPUB is not finished just because it opens and roughly conveys meaning. It must preserve the source, expose uncertainty honestly, and read as a finished Chinese book.

## 精修重点 / Refinement Focus

### 1. 底本 / Base Text

- 翻译必须从古希腊文底本出发。
- 每个项目必须有 `metadata/source_witness_manifest.md`。
- 扫描、OCR、TEI/XML、网页转写、校勘本、参考译本要分清来源类型。
- 未校 OCR 不能直接批量翻译。
- 异文、残损、拟补、语法歧义必须进入 `qa/textual/textual_uncertainty_log.md`。

### 2. 标题 / Titles

- 必须区分原作者题名、校勘者题名、现代编辑目录、参考译本目录和 AI 概括。
- 需要时建立 `metadata/chapter_title_map.yaml`，包含 `source_full`、`source_type`、`nav_title`、`display_title`、`subtitle`、`title_note`。
- `nav.xhtml` 使用短目录题名，页面内再用主标题和副标题承载完整信息。
- 如果原文只有卷号、书号、节号、命题编号或简单题名，不得为中文 EPUB 自创读者可见小标题；解释性概括只能放入 `title_note`、制作说明或 QA 记录。
- 标题中的专名只用中文译名，不放古希腊文原名、拉丁化转写或外文括注。

### 3. 句子 / Sentences

- 先保事实、论证和语法关系，再调中文。
- 长周期句不应硬搬成中文长串；可按中文阅读顺序拆分，但必须保留因果、转折、递进和让步关系。
- 分词链必须判断功能：时间、原因、条件、让步、结果或伴随动作。
- 省略主语、宾语或指代时，中文可以补足理解所需关系，但不得补出原文没有的事实。
- 被动、无人称表达和抽象名词结构应转成自然中文，同时保留论证重心。
- 古代技术术语常比现代中文更简短。翻译时不得把简短原文硬译成意义不明的中文，例如“割某线”“超过某线”这类读者不易理解的作图语，应译出现代读者能直接看懂的几何关系：交于何线、落在何处、是否在延长线上、是否为半径或弦。
- 现代化改写不是加长解释。若一句原文只是简洁作图，应译成简洁现代中文；避免把一个作图动作扩写成多句说明，除非原证明链需要显式补出关系。
- 技术证明中的可读性标准是“读者能顺着图和点名复原关系”，不是“逐词贴合古希腊语序”。凡译文保留了原证明事实、点名、线段和依赖关系，就应优先采用自然中文。

### 4. 段落 / Paragraphs

- 超过 300 字的普通中文段落应进入专门复核；超过 500 字通常要判断是否拆段。
- 拆段不能破坏原文论证、叙事阶段或证明步骤。
- 对技术文本，段落拆分必须保持定义、命题、证明、结论的层次。

### 5. 专名、术语与译注 / Names, Terms, and Notes

- 人名、地名、神名、民族名、学派名、书名、星座名和技术术语必须统一。
- 有权威中文译名时优先使用；没有时建立本书转写策略。
- 正文首次自然出现陌生专名时，可在译注或括注记录古希腊文原名或拉丁化转写。
- 标题、目录、页眉、图注索引等标题性位置不计入正文首次出现。
- 译注用于解释异文、古代制度、术语传统、现代读者可能误解的概念；不得替代正文翻译。
- 对读者反复遇到的技术依据，例如《几何原本》卷号、命题号、定义或系，正文可用小号依据标记保留引用；章末集中注释应说明该依据的大意，使读者不必另查原书才能理解本章证明。

### 6. 参考译本 / Reference Translations

- 第二语言译本只能作为 reference witness。
- 参考译本影响判断时，只能记录为校读线索、差异说明或译注依据。
- 不得复制仍受版权保护参考译本的表达、注释、图表、表格或结构。
- 参考译本与底本冲突时，必须回到古希腊文底本和校勘记录。

### 7. 标点和排版 / Punctuation and Typography

- 普通中文正文不得出现 ASCII 分号 `;`。
- 中文分号 `；` 只能用于真实并列分层，不得机械对应希腊文连接词。
- 中文字符之间不得保留用于纸书对齐的连续空格。
- 扫描本 running title、页眉、页脚、页码、校勘页边标记不得混入正文。
- 附录、索引、表格、页码目录等旧纸书结构，必须转成适合 EPUB 的列表、表格、注释或导航结构。
- 旧纸书正文中的 `* * * * *`、`*****`、`----` 等分隔符不得进入成书正文；若只是排版分隔，直接删除；若确实表示内容转换，应由段落自然承接或另写小标题。

## 本书目标文档位置 / Book Goal Location

如果某本书已经发现标题、段落、术语、译注、排版、底本、异文或参考译本边界方面的系统问题，目标文档必须放在：

```text
books/zh-Hans/{number}_{book_id_slug}/goal/
```

不能放在仓库根目录的通用 `goal/` 下。

## 模板回填 / Template Backfill

古希腊文到简体中文项目中的可复用经验必须回填到三层：

1. `books/zh-Hans/{number}_{book_id_slug}/goal/`：记录这本书的具体问题和执行计划。
2. `template/epub_pipeline/common/`：记录所有语言方向都适用的 EPUB、标题、QA、路径和流程规则。
3. `template/epub_pipeline/grc-zh-Hans/`：记录古希腊文源文到简体中文的专用问题，例如底本、异文、转写、参考译本边界和古希腊文句法干扰。

如果启用特殊 profile，还要把可复用经验回填到对应 `profiles/`。
