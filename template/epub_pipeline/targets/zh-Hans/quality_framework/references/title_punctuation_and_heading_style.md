# 简体中文标题标点与标题层级规则

本文件适用于所有译入简体中文的书，不限源语言。

## 总原则

中文成书标题不是原文标点的逐字复制。标题应同时满足：

- 准确传达原文题名功能。
- 在中文里自然、利落、可记。
- 在手机 EPUB 中不压迫正文。
- 在目录中容易扫读。

## 破折号处理

- 原文用多个破折号串联小题时，中文不应机械复制成多个 `——`。
- 一个中文章节标题中原则上不超过一个 `——`。如果超过一个，应优先改为冒号、顿号、逗号、副标题或标题说明。
- 用 `：` 表示“主标题 + 说明性副标题”通常比多个 `——` 更适合中文 EPUB。
- 多个并列小题可改为副标题行，或在标题说明中用顿号、逗号处理。

示例：

```text
不推荐：第五章 制作皮里式雪橇——在北极之夜狩猎——容易激动的狗
推荐：第五章 制作皮里式雪橇
副标题：北极之夜的狩猎与躁动的狗群
```

## 标题长度

- EPUB 目录题名应短于页面显示题名。
- 页面主标题宜控制在 18 个汉字左右；超过 24 个汉字时必须考虑拆分。
- 副标题可以较长，但不应与主标题同等字号并挤在同一行。

## 标题中的专名

- 中文读者已有稳定译名的专名，应优先用中文译名。
- 首次出现的重要外文专名，可在正文、译注或术语表中说明，不建议堆进章节主标题。
- 船名、地名、人名如需保留原文，应确认是否会显著拉长标题。陌生人名、陌生地名不必强制汉译，例如 `Professor Marvin`、`Harry Whitney`、`Grant Land` 可保留英文；若标题或正文选择汉译，首次出现应保留英文原名。

## 标题门禁

章节进入 `chapters/final/` 前，必须检查：

- 是否存在半截标题或被截断标题。
- 是否存在重复标题、页眉题名、下一章题名或书名误入正文。
- 是否存在多个 `——` 串联。
- 是否存在“标题即目录摘要”的纸书痕迹。
- 是否能拆成“主标题 + 副标题/说明”。
- 是否需要在 EPUB nav 使用短题名，在章节页显示完整题名。

未完成上述处理，不得把标题问题留到 EPUB 构建阶段。

## 附录与名单呈现

- 简体中文 EPUB 中，纯姓名表、目录表、索引或原始拼写清单不应被强行改写为普通中文叙述段。
- 如果保留等宽预格式化块，应确认手机窄屏可换行、不会横向溢出，并在 QA 记录中说明保留理由。
- 如果改为列表或表格，应确认构建脚本能生成有效 EPUB XHTML，并保留原始拼写、星号、重名和排序。
