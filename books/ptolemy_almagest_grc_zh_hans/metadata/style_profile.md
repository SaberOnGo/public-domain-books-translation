# 《Almagest》文体与技术表达画像 / Style Profile

style_profile_status: `PASS_FOR_PRETRANSLATION_PLANNING_ONLY`

## 中文目标

清楚、稳健、可校验。译文应像严谨中文科学史译本，而不是文学化散文或现代科普改写。

## 叙述声音

- 主声调：冷静、证明式、技术说明式。
- 次声调：必要处用译注解释古代模型和现代读者误解风险。
- 禁止腔调：现代科普爽文、文学化抒情、过度简化的说明书、现代天体物理改写。

## 句法策略

- 长周期句：拆成中文句群，但保留证明关系。
- 证明句：明确“设/作/由此/所以/证得”的逻辑；不得为了贴近原文而写出难懂的套叠句。
- 几何证明优先使用现代中文可读表达，例如“某弧对应的弦”“半弧对应的弦”；原文直译式表达如“所对弧之半所对的直线”只能出现在注释或术语说明中。
- 作图动作优先译成清楚、简洁的现代几何关系：例如 `以 Δ 为圆心、ΔΕ 为半径作圆，使它交 ΑΔ 于 Η，并在 ΔΖ 的延长线上交于 Θ`。不得硬译成“将割 ΑΔ，并超过 ΔΖ”，也不得扩写成冗长解释。
- 表格说明句：保留字段和数值结构。
- 图表引用句：保持图号、点名、线名、圆名一致。

## 术语策略

- 模型术语先锁定再翻译。
- 数学动作词全书统一。
- 古希腊原词和必要转写进入术语表，不塞入标题。
- 现代术语仅作注释帮助，不覆盖古代概念。

## 注释策略

- 解释古代模型和现代术语差异。
- 解释关键数学步骤。
- 解释参考译本差异。
- 解释图表重绘与原图关系。
- EPUB 读者版优先使用章末集中注释：正文用数字标号提示，章末列出注释，避免手机阅读时脚注挤压正文。
- 古代术语若直译影响理解，正文采用现代可读术语，章末说明原文表达和古今概念边界。
- 《几何原本》依据在正文中用小号“依据《几何原本》...〔n〕”标记；章末集中说明每条依据的大意，不让读者必须另查《几何原本》才能理解证明。

## 数值显示策略

- 正文和 EPUB 可见文字不得裸用“整数部加小分的内部六十进制记法”。
- 角度写作 `1°30′`、`0°45′`。
- 弦长等非角度六十进制量在读者版正文中写作 `37p04′55″` 这类格式；`p` 表示整数部/单位部，`′` 和 `″` 表示六十进制小分，不是角分秒。
- 不得写 `37份4′55″` 或 `1375份4′15″`。`份` 只用于解释“直径取 120 个等分单位”的背景，不作为连续数值记法。
- 不得把非角度六十进制数统一改成十进制小数；十进制只能作为注释或校验说明。

## 风格红线

- 不得为了流畅省略证明步骤。
- 不得把表格散文化。
- 不得把古代天文学改写成现代天文学。
- 不得直接转译 Toomer 或其他现代译本。
- 不得让 GPT-Image-2 图直接成为最终学术图。
- 不得把读者不懂的内部数值记法直接暴露在正文中。
- 不得把古典几何作图语硬译成现代中文中意义不明的动词短语；也不得为了说明一个点位关系而过度解释。
