# 翻译测试协议

## 目的

用软件工程的测试思想检验翻译方法，而不是凭感觉批量生产。

## 测试集

测试集由三类组成：

- 公版优秀译文：可存入仓库，用于长期回归测试。
- 用户自备合法片段：例如仍在版权期的经典译本，只能用户本地提供，不能把长段原文或译文写入仓库。
- 当前待翻书样本：从本书抽取难段。

## 私有优秀译本样本

当前可使用的私有基准：

- 杜拉斯《情人》开头 / 王道乾译本：见 `tests/private_benchmark_cases/duras_wang_opening_benchmark.md`。

这个样本只用于提炼“优秀中文译文如何成立”的方法，不用于复刻其句子，也不要求待翻书模仿它的抒情小说腔调。

## 关于《情人》等仍在版权期作品

可以作为方法验证的“私有对照样本”，但必须满足：

- 用户自行提供合法持有的短片段。
- 本仓库不保存长段原文或长段译文。
- 输出不公开复刻受版权保护译文。
- 只记录评分、问题类型、方法结论。

## 测试流程

1. 选 3-5 段参考样本。
2. AI 先按本框架翻译。
3. 与优秀人译做维度对比：
   - 语义准确度
   - 中文节奏
   - 意象处理
   - 句子收束
   - 情绪强度
   - 可记忆度
4. 记录失败句型。
5. 修改 prompt 与规则。
6. 重测，直到样本通过。

## 当前书必须增加的测试

每本正式开工前，必须从待翻书中抽取 3 段做同样测试：

- 开篇定调段。
- 高现场感段落。
- 含历史称谓、专业术语或文化负载的段落。

如果这 3 段中任一段 FAIL，不得进入批量章节翻译。

## PASS 标准

- 准确性 5/5。
- 中文性 4/5 以上。
- 风格 4/5 以上。
- 叙事节奏 4/5 以上。
- 不出现“读得懂但无味”的核心段落。

## 回归测试

每次修改翻译 prompt 后，重新测试同一批样本。若新版本比旧版本更直、更平、更硬，则回滚规则。
